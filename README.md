# MiniLove2D
when i realised people on mobile couldnt instantly run their love2d code, i decided that if no one fixes this problem, I will fix the problem, basically love2d with autocomplete and two outputs, one to see what ur love2d code made, one to see ur errors n stuff.

## How to Use MiniLove2D

Getting started with MiniLove2D is super straightforward, even if you're on mobile.

### The Basics
1. **Fire up the editor** - Open the main interface and you'll see a code editor on one side. This is where you write your Love2D code, just like you normally would.

2. **Write your code** - Type out your Lua code like you're used to. The cool part? You get **autocomplete** as you type, so you don't have to memorize everything. Just start typing and suggestions should pop up.

3. **Run it** - Hit the run button (or however you submit it on your version), and watch two things happen at once:
   - **Output panel**: Shows you what your code actually created - your game or animation or whatever you're building
   - **Error panel**: If something went wrong, you'll see all the errors and debug info right there instead of wondering what's broken

### Pro Tips
- If you mess up the code, just fix it and run again. The error panel will instantly update so you know what you need to fix.
- Since this runs instantly on mobile, you can actually prototype and test your Love2D projects on the go without needing a full dev setup.
- The autocomplete is there to help you remember function names - lean on it, especially if you're learning Love2D for the first time.

### That's It!
Seriously, that's the whole workflow. Write → Run → See results OR see errors → Fix → Repeat. Way easier than trying to get Love2D set up on your phone before.
